# Security Policy

Please report suspected vulnerabilities privately to the repository maintainer when a private reporting channel is available. Otherwise, open a minimal issue that contains no secrets, personal data, LinkedIn content, or exploitable detail and ask for a private contact channel.

Particularly relevant reports include credential exposure, permission-scope expansion, unauthorized provider requests, retained raw content, identity de-identification failures, prompt-boundary bypasses that expose extension data, and unintended LinkedIn actions.

Never include a live Gemini API key in logs, screenshots, fixtures, issues, or pull requests. Revoke any key that may have been exposed.
