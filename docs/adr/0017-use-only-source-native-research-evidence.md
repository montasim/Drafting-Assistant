# Use only source-native research evidence

Public Source Research will collect bounded Source Evidence only from each approved source's official public API or RSS feed and will never follow or scrape arbitrary linked pages. Starting a search runs all enabled source adapters concurrently with the user's saved topics; their responses are validated, normalized, and deduplicated locally before one provider-neutral synthesis job ranks the combined evidence and creates the Idea cards. A sourced search returns at most one qualifying Idea per enabled source—currently up to five total—and a weak, unavailable, or cross-source duplicate candidate contributes no filler card. The latest result set forms an Idea Search Session in `chrome.storage.session`: it survives extension-tab navigation, panel closure and service-worker suspension, but never enters Work History merely by being displayed and is cleared by Chrome restart, extension reload or update, or replacement by **New search** without confirmation. A candidate that successfully creates a post stores the resulting Work History record UUID in that session; returning through **Back to ideas** and selecting the same candidate reopens its existing editable post without another provider call, while **Regenerate** creates a revision. A later search produces new candidate UUIDs and may therefore create a distinct record from similar evidence; posts already created remain in Work History when the temporary set is replaced. Gemini receives only the validated evidence, and Groq receives the same evidence only through the approved fallback path. Providers are not asked to browse, select additional websites, invent citations, or substitute their own memory for source evidence. Candidates without enough source-native evidence are skipped rather than turned into headline-based Ideas. Every surfaced sourced Idea retains a clickable Source Reference to the original page for manual verification, and one unavailable source does not discard valid results from others: the completed UI identifies the partial source failure rather than fabricating replacement provenance. If no source produces a qualifying Idea, the workflow shows the single approved Experience Fallback rather than fabricating a sourced card or generating an unsourced Idea list. Its unfinished lesson autosaves only to `chrome.storage.session`, survives extension navigation, panel closure and service-worker suspension, and does not enter Work History until post generation succeeds. A successful post replaces that lesson draft with a temporary reference to its Work History UUID; returning to the Idea tab shows the completed lesson with **Continue editing**, which reopens the existing record without a provider call, while **Regenerate** inside the editor creates a revision. Starting a new search or explicitly clearing the lesson removes that temporary reference without deleting its History record. Chrome restart or extension reload or update clears the lesson draft and temporary reference, failure or cancellation preserves the unfinished lesson, and incognito lesson state remains confined to its incognito session.
