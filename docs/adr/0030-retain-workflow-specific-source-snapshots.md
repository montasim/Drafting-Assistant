# Retain workflow-specific source snapshots

Thoughtline will keep only the source material needed to understand and resume each Work History entry. Reply records retain author, optional verified canonical-link metadata, bounded post and Response Target excerpts, summaries, four generated directions, edits, feedback, and timestamps without archiving the complete Visible Discussion; a missing LinkedIn permalink does not prevent retention and is represented as **Link unavailable**. Sourced Idea records require bounded Source Evidence and a clickable Source Reference without copying an external article; Experience Fallback posts use the same **Ideas** History filter but retain the user's supplied lesson, an experience badge, generated and edited post, revisions, feedback, provider provenance, and timestamps without an external link or fabricated source summary. Rewrite records retain the full user-pasted original and selected rewrite goal together with generated and edited results so comparison, record-boundary checks, and regeneration remain possible. Wherever the approved interface provides English and Bangla summary tabs, both summaries are generated and validated in the original Foreground AI Job, stored together, and switched locally without another provider request. These History Source Snapshots follow retention, deletion, Clear All, storage-quota, trusted-context, and incognito rules and never contain raw DOM, HTML, or provider request envelopes. The approved prototype's Idea labels remain unchanged.
