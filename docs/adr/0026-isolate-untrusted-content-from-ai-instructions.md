# Isolate untrusted content from AI instructions

Thoughtline will treat LinkedIn content, public-source evidence, imported profile text, and user-pasted drafts as untrusted data rather than instructions. Before a provider call, the application will build an Untrusted Content Envelope from allowlisted fields, validate it with Zod, normalize text and non-semantic Unicode/control characters, enforce per-field, collection, and total-context limits, and reject invalid or oversized input without silently truncating or repairing it. Human-visible LinkedIn names may remain because the user explicitly approved them, while Profile Import will remove contact details locally before transmission. The provider adapter will serialize the envelope as labeled JSON separated from the trusted system instruction, tell the model never to follow embedded instructions, and expose no browser, Chrome, network, storage, or publishing tools. Provider-native structured output constraints will be followed by local Zod validation; generated values will be handled only as editable plain text, never executed or rendered as trusted HTML, and never posted automatically. Adversarial tests will cover direct and indirect prompt injection, hidden or malformed DOM content, control characters, oversized input, extra output properties, invalid schemas, and attempted instruction leakage.
