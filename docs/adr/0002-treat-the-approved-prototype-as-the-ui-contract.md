# Treat the approved prototype as the UI contract

The highest Approved Prototype Version defined by ADR 0038 is the visual source of truth for the production extension: color, typography, icons, spacing, component states, and responsive side-panel behavior must be reproduced rather than redesigned. Production UI will be composed from reusable Tailwind CSS and shadcn/ui components, with design tokens centralized in the Tailwind theme and no handwritten component-level vanilla CSS. Approved behavioral truth takes precedence over fixture copy: the Settings readiness chip **Evergreen ideas** becomes **Ideas**, while the bottom-navigation label remains exactly **Idea**; other documented copy corrections for passive extraction, automatic provider fallback, provider roles, History filters, consent, confirmation dialogs, and Schedule Preview follow the same rule without authorizing a visual redesign.
