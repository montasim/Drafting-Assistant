# Distribute through GitHub Releases

Thoughtline will initially be distributed to a technical/private audience as versioned GitHub Release ZIP artifacts and may be published through the Chrome Web Store later. The project will generate and retain its own stable release identity, place only the corresponding public key in the extension manifest, and keep any private signing material outside the repository; this avoids reserving a Web Store item while keeping the unpacked extension ID stable across release builds. Preview users accept Chrome Developer Mode, Load unpacked installation, and manual in-place updates from a stable local directory; the repository README will be the installation and operating guide, use screenshots captured from the production extension, document checksum verification, backup precautions, replacement and reload steps, and warn that a later Web Store identity may require an explicit migration rather than silently presenting a new empty installation.
