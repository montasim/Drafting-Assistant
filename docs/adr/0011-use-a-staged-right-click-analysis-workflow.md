# Use a staged right-click analysis workflow

After LinkedIn page permission is granted, Thoughtline registers one Chrome-native item named **Draft a reply with Thoughtline** for page, selection, link, and image context clicks on `https://www.linkedin.com/*`; Chrome controls the menu's visual styling, and the extension validates that the remembered DOM target belongs to a supported post, comment, or reply after selection. Every context-menu click creates a distinct Analysis Request: it remembers the exact DOM target, opens the side panel, verifies setup and permissions, extracts and validates one Post Context, publishes staged progress, executes the provider-neutral reply use case with Gemini and Cross-Provider Fallback, validates the structured result, and persists current work plus a new bounded Reply History entry. Thoughtline will not merge separate requests by LinkedIn URL, author, or excerpt because the target, visible discussion, settings, or writing profile may have changed; only **Regenerate selected reply** adds a revision to the selected direction in the current record. Thoughtline will improve the reference `/ai-comments` flow with cancellable requests, dedicated recovery states, richer source/history metadata, typed message contracts, request-correlated state, and targeted storage subscriptions while preserving passive extraction and fail-closed validation.
