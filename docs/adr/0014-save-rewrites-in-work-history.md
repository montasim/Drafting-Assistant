# Save rewrites in work history

Completed Rewrite Drafts will be retained in Work History as a third type alongside Replies and Ideas. A successful Reply automatically creates one record containing its summary and four directions, and a successful Generate job automatically records its original, rewrite goal, and rewrite; Idea-search candidates remain temporary and create History only after the user generates a post from one. Unsubmitted Generate source text and its selected goal autosave only to `chrome.storage.session`, surviving extension navigation, panel closure and service-worker suspension without entering Work History; explicit clearing, Chrome restart, extension reload or update, or replacement by a successful generated record removes that compose state, while failure or cancellation preserves it. Incognito compose state remains confined to its incognito session. Debounced text edits and immediate feedback updates modify a completed record, while regeneration with the same source and goal appends a revision rather than creating an unrelated duplicate. If **Edit source** changes either the original content or rewrite goal, the next successful generation creates a new Rewrite record and leaves the previous record unchanged; abandoning changed input creates nothing. Each generated Reply direction, Rewrite, or Idea-derived post retains the current value and at most five prior unpinned generated revisions. Users may pin revisions, which are exempt from automatic pruning and do not count toward that limit; when the limit is exceeded, the oldest unpinned generated revision is removed. Pinning does not make data undeletable: a pinned revision is still removed by its confirmed manual deletion, deletion of its parent record, confirmed Clear All, or retention expiry, and it participates normally in export and import. Restoring a revision is a local, immediate save that does not start an AI job: unless identical content is already retained, the displaced current value first becomes a prior revision, then the selected revision becomes current without changing the selected revision's pin state. Ordinary typing updates the current edited value and does not create per-keystroke revisions. Revision management is available only inside a Work History record through a collapsed **Revisions** disclosure that appears when prior revisions exist; the active Reply and Generate screens remain unchanged. Each revision presents a short preview, timestamp, provider provenance, current or pinned status, and copy, restore, pin or unpin, and confirmed delete actions. History search is an immediate local Unicode-aware substring match after the **All work / Replies / Rewrites / Ideas** filter: it searches current visible metadata and content—Reply author, excerpt, and four current versions; Rewrite original and current rewrite; Idea title, source metadata or lesson, and current post—without AI, fuzzy inference, or matches found only in hidden prior revisions. Retention counts parent records rather than revisions and orders them by `updatedAt`, newest first. Generation, manual edits, feedback, regeneration, and revision restoration are meaningful updates; viewing, searching, and copying are not. **Keep for 30 days** measures from that last meaningful update, while count-based policies keep the latest 20 or 50 parent records. Choosing a stricter policy previews the number of records it will remove and requires confirmation before pruning; pinned revisions do not exempt their parent record. Deleting an entire History record requires an AlertDialog that names the record, explains that its source snapshot, drafts, edits, feedback, and revisions will be removed, and places initial focus on **Cancel**; confirmed Clear All follows the same default-safe pattern. Failed or cancelled jobs create nothing, storage failure follows the shared recovery policy, and incognito work remains ephemeral. Individual deletion, confirmed clear-all, and the approved retention choices apply consistently while each type preserves its workflow-specific source, revisions, edits, feedback, provider provenance, and timestamps.
