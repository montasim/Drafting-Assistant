# Keep incognito work out of persistent history

Thoughtline may be enabled by the user in Chrome's split-incognito mode, but Reply, Generate, and Idea work created there will remain ephemeral and will never be added to persistent Work History. The incognito context may read the user's existing settings, writing profile, and encrypted Provider Credentials and may keep the current result only in incognito-scoped memory for review, editing, and copying; closing that context discards the work. Ratings, edits, and selections may retain their visual state for that incognito session but will not create Feedback Evidence or change Learned Writing Preferences. This boundary is enforced by the application because Chrome shares `storage.local` between regular and split-incognito processes.
