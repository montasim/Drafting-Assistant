# Reuse the proven passive extraction behavior

Thoughtline will adapt the `/ai-comments` right-click targeting and LinkedIn extraction mechanism in the new codebase: remember the exact context-menu target, resolve one containing supported post, collect human-visible data already rendered inside that post's DOM, validate the complete Post Context, and fail closed on unsupported layouts. A post Response Target includes every already-rendered visible comment and reply inside that post; a top-level comment target includes only that comment and its visible replies; a nested-reply target includes its visible parent comment and the visible replies in that same thread. Unrelated top-level threads are excluded for comment and reply targets, and a nested reply without a visible parent fails closed rather than sending incomplete context. Visible author and participant names may be included in provider analysis, while participant profile URLs are not required. Thoughtline retains a canonical post permalink only when one is already rendered and validates, or uses the current URL on a valid post-detail page; it never clicks, opens menus, or requests LinkedIn data to discover one. Valid text may still be analyzed without a permalink, in which case Reply and History show **Link unavailable** instead of linking to the general feed. The extension will not scan unrelated feed posts, request hidden LinkedIn content, click, scroll, expand content, mutate the page, or automate engagement. Because names are intentionally retained, the prototype progress label **Names and profile links removed** will be corrected in production to **Visible discussion validated** without changing the approved layout.
