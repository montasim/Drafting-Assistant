# Restore the active workspace within a Chrome session

Thoughtline will restore the Active Workspace when its side panel closes and reopens during the same Chrome session. Session state retains the active bottom-navigation tab, workflow screen, and—when the screen represents completed work—only its Work History record UUID; completed content is reloaded from Work History rather than duplicated in session storage. Temporary Idea results and unfinished Generate or Experience inputs continue to use their separately defined session state. If a referenced record was deleted, expired, or cannot be loaded, Thoughtline opens that tab's safe default screen instead of presenting stale content. Chrome restart, extension reload, or extension update clears the Active Workspace and opens the default Reply screen; incognito workspace state remains confined to its incognito session.
