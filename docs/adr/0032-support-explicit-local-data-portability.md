# Support explicit local data portability

Thoughtline will provide manual Data Archive export and import under **History & storage** so a local-first, GitHub-distributed installation has a user-controlled recovery and device-migration path without cloud synchronization. The existing Settings section will gain right-aligned secondary **Export data** and **Import data** buttons; each opens a focused dialog for passphrase entry, category selection, and import or export preview, while readable JSON remains under an **Advanced** disclosure inside the export dialog to keep the primary panel compact. The versioned archive will contain Work History, History Source Snapshots, the writing profile, settings, and Learned Writing Preferences while structurally excluding Provider Credentials, encryption keys, Foreground AI Job state, diagnostics, permissions, and consent records. Internal records and archives will receive Web Crypto UUIDs at creation and retain those IDs through export and import so accidental collisions are negligible and repeated imports can be deduplicated. Export will produce a passphrase-encrypted `.thoughtline` archive by default using versioned Web Crypto parameters; the passphrase is never retained or recoverable, and an advanced readable-JSON export requires a specific privacy warning. Import will validate the outer envelope, decrypt locally when applicable, parse the entire payload through Zod, reject unsupported or malformed files without partial mutation, and preview included record counts and profile/settings effects. Merge preserves UUIDs, adds unseen records, and resolves a duplicate UUID with the valid record having the newer `updatedAt`; profile, settings, and learned preferences change only when their preview checkboxes are selected. Confirmed replacement affects only selected categories, and either mode completes as one atomic write or leaves existing data untouched.
