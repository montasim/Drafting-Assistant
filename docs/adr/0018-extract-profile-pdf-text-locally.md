# Extract profile PDF text locally

Profile Import will parse the user-selected LinkedIn PDF inside the extension, remove contact details, validate and bound the extracted professional text, and pass that text through the provider-neutral profile capability with Gemini primary and Groq fallback. The ownership confirmation starts unchecked and is required only after a PDF is selected; it blocks the import action rather than onboarding, resets when the file is removed or processing finishes, and is irrelevant when the user completes profile fields manually. A successful result opens an editable preview of the proposed role, expertise, and audience and requires an explicit **Apply changes** action before replacing any saved profile fields; cancellation, validation failure, provider failure, or panel closure leaves the existing profile untouched. The raw PDF is never persisted and is discarded immediately after local extraction and request preparation, avoiding provider-specific file APIs and keeping fallback behavior consistent.
