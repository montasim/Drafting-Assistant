# Allow one active foreground AI job

Thoughtline will permit only one active Foreground AI Job across Reply analysis, Generate rewriting, Idea research, post generation, profile derivation, style analysis, onboarding credential validation, and **Check connections** in all normal and incognito extension contexts. A connection-validation job may run its fixed Gemini and Groq tests concurrently inside that single lock, but it cannot overlap another provider job. Global coordination uses one minimal `activeJob` lease in shared `chrome.storage.local`, containing only a random owner UUID and fixed expiry with no content, URL, workflow name, provider response, or credential. A candidate writes its UUID, confirms ownership before starting, and blocks if another unexpired owner exists; the owner removes the matching lease on completion, cancellation, or panel closure, while a short hard job timeout makes an abandoned lease self-expire without a heartbeat system. Each job captures an immutable settings, profile, preference, source-selection, and credential-version snapshot at start; later Settings edits persist for the next job without changing the active request. Foreground provider requests are owned and executed by the initiating side-panel extension page rather than relying on a long service-worker fetch: internal Thoughtline navigation and browser-tab changes do not cancel the job, but closing the owning panel aborts its request, releases the lock, and creates no incomplete History record. An attempted competing job shows that another activity is running and may cancel only a job owned by its current extension context, preventing free-tier contention, duplicate provider charges, overwritten state, and ambiguous History records. Navigation, editing, copying, History browsing, unrelated-record deletion, and read-only Data Archive export remain available. Archive import, credential replacement or removal, Clear All, retention changes that would prune records, and deletion of a record currently being regenerated require cancellation first; revoking AI Processing Consent cancels the active request immediately before recording the revocation. Scheduled Idea Searches run independently in the future external service.
