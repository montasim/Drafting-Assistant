# Request extended storage only for unbounded history

Thoughtline will use Chrome's normal local-storage quota for its default 20-record, optional 50-record, and 30-days-since-last-meaningful-update Work History policies; revisions do not count as separate records. The `unlimitedStorage` permission will be declared as optional and requested with an explanation only when the user explicitly selects **Keep until deleted**; denial leaves the previous bounded policy active, and later revocation returns the user to a bounded policy without silently deleting existing records. Selecting any stricter policy first previews the number of affected records and requires confirmation before pruning. Extended storage does not remove application-level Zod limits, byte accounting, trusted-context access, storage-full recovery, or protections against runaway writes.
