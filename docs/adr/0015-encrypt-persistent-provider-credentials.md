# Encrypt persistent provider credentials

Gemini and Groq Provider Credentials will persist across Chrome restarts as purpose-bound AES-256-GCM ciphertext in `chrome.storage.local`, unlocked by a non-exportable device key retained in IndexedDB and exposed as plaintext only through trusted extension contexts for active use. Decrypted session copies may live in `chrome.storage.session`; credentials are excluded from logs and diagnostics and can be revealed, validated, replaced, or removed by the user. Key fields remain masked by default and decrypt for display only after an explicit eye-button action; they remask when the field loses focus, its disclosure closes, or the side panel closes, and plaintext is excluded from logs, diagnostics, form-state snapshots, accessibility labels, and persisted UI state. Both credentials must validate for Provider Readiness, and removing or invalidating either blocks new Foreground AI Jobs until the pair is restored.
