# Preserve work across recoverable failures

Thoughtline will use a shared recovery policy across every Foreground AI Job: preserve user input after failure or cancellation, apply Cross-Provider Fallback only for its approved Gemini-specific triggers, disclose a successful provider switch without interrupting the result, combine dual-provider failure into a safe recovery state, and keep unsaved generated output editable and copyable after a storage failure. Partial public-source failure may return valid Ideas while naming unavailable sources; invalid, unreadable, and oversized inputs remain distinct failures. Every state will offer the relevant retry, credential, permission, input-size, or storage action, while UI errors and the Diagnostic Bundle exclude credentials, source content, and raw provider responses.
