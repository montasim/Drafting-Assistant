# Version UI prototypes as immutable snapshots

Thoughtline UI prototypes will be retained as immutable, numbered snapshots rather than edited in place. The existing `prototypes/simplified-ui-original.html` is the preserved original baseline, and the specification-review update begins `prototypes/simplified-ui-v2.html`; every later UI change must create the next sequential file such as `simplified-ui-v3.html`, include its version visibly in the prototype, and add a concise entry to `prototypes/README.md`. No prior prototype may be overwritten, renamed, or deleted merely because a newer design exists. The highest Approved Prototype Version explicitly accepted by the user becomes the visual source of truth referenced by ADR 0002 and by production visual-regression tests, while earlier versions remain comparison and decision-history artifacts.
