# Enforce automated quality gates

Thoughtline will treat formatting, static analysis, unit behavior, extension integration, accessibility, and visual parity as release gates rather than optional cleanup. The repository will use strict TypeScript, ESLint, Prettier, `.editorconfig`, Husky with lint-staged for fast pre-commit checks, Vitest and React Testing Library for domain, adapter, hook, and component tests, and Playwright for onboarding, side-panel workflows, Chrome-extension integration, accessibility smoke tests, and screenshot comparison at the canonical viewport and responsive bounds. Provider and public-source adapters will be contract-tested with deterministic fixtures rather than live credentials in CI; LinkedIn extraction tests will use representative sanitized DOM fixtures, and security tests will cover Zod rejection, message authorization, prompt injection, storage migration, credential redaction, incognito boundaries, and concurrent job claims. Pull requests and GitHub Release builds must pass formatting checks, lint, typecheck, unit/integration tests, production build, and designated end-to-end and visual suites; Husky improves local feedback but CI remains authoritative and cannot be bypassed by skipping hooks.
